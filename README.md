# CU Student Bank UI

## 1. Aim
To build a responsive banking interface with deposit and withdrawal functionality using HTML, CSS, and JavaScript.

## 2. Objective

- Design a balance display using **CSS Flexbox**
- Implement a **responsive layout** using media queries
- Create **Deposit** and **Withdraw** buttons with JavaScript event handlers
- Perform **input validation** and display user feedback messages
- Update balance dynamically using **DOM manipulation**

## 3. Hardware / Software Requirements

### Hardware Requirements
- Computer or Laptop
- Minimum 4 GB RAM
- Internet connection (optional)

### Software Requirements
- Any modern web browser (Chrome, Edge, Firefox)
- Code editor (VS Code / Sublime / Notepad++)
- Operating System: Windows / macOS / Linux  

*(Same as Experiment 1)*

## 4. About the Program

This program demonstrates the implementation of a simple banking system interface for students.  
It allows users to:

- View their current balance
- Deposit money
- Withdraw money with validation
- Receive instant success or error messages

The project focuses on:
- **Responsive web design**
- **CSS Flexbox layout**
- **JavaScript DOM manipulation**
- **State management using variables**

This lab helps students understand how frontend technologies work together to create interactive and user-friendly web applications.

## 5. Technologies Used

- **HTML5** – Structure of the application
- **CSS3** – Styling and responsiveness
- **JavaScript (ES6)** – Logic, validation, and interactivity

## 6. Outcome

After completing this experiment, students will be able to:
- Design responsive user interfaces
- Handle user input efficiently
- Manipulate the DOM dynamically
- Implement basic business logic using JavaScript

## 7. Conclusion

The CU Student Bank UI project successfully demonstrates a responsive and interactive banking interface.  
It serves as a practical example for understanding frontend development concepts and real-time user interaction.
